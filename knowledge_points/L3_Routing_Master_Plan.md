# L3 路由核心考点突击总计划 🗺️

基于对最新 INHN0012 (2023-2025) 考卷以及老课程 GRNVS 题库的地毯式扫描，**网络层 (Layer 3) 的路由机制**是整门课中**分数占比最高、失分最惨重**的绝对核心。

老教材非常喜欢考 Dijkstra (迪杰斯特拉) 算法的画图推演。但在最新的 **INHN0012** 考试体系中，教授的命题风向发生了巨变，他们现在把考点的重头戏死死砸在了 **RIP (Distance Vector 距离矢量路由)**、**静态路由大表连线**、**NAT 地址转换**，以及极其硬核的 **IP Subnetting 子网切割**上。

为了确保这 15-20 分万无一失，我们将按照考频和实战杀伤力，分四个阶段逐一击破这块最难啃的骨头：

---

## 第一战：动态路由协议 (RIP 与 距离矢量的终极死穴)
**考试权重**：极高 (通常是一道 15-20 分的独立压轴大题，如 2024 Endterm)
- **核心概念**：RIP 协议脑子是一根筋，它只认“跳数 (Hop Count)”（最大 15 跳）。这种“盲人摸象”式的通信会导致严重的“Count-to-Infinity (无限计数踢皮球)”网络灾难。
- **我们的任务**：创建 `L3_RIP_and_Distance_Vector.md` (已完成 ✅)。
  - 解析路由表互相传递了什么信息。
  - 彻底拆解 **Count-to-Infinity** 灾难发生的剧情。
  - 牢背它的三个救命解药：**Split Horizon (水平分割)**, **Poison Reverse (毒性逆转)**, 和 **Triggered Updates (触发更新)**。

## 第二战：IP Subnetting 子网切割与最长前缀匹配 (LPM)
**考试权重**：极高 (直接拉开分数差距的数字杀手局)
- **核心概念**：给一个大网段（比如 `10.0.1.0/24`），让你手工切割、分配给楼里的不同住户，计算广播地址。并推演当一个 IP 发过去时，路由器是遵循什么规矩把它塞进转发表的（`/XX` 匹配的艺术）。
- **我们的任务**：创建 `L3_IP_Subnetting_and_LPM.md` (即将进行 🎯)。
  - 提供一套傻瓜式的 `/24` 到 `/30` 掩码换算推盘心法。
  - 拿真实的期末路由表试卷，手动走一遍 **LPM (最长前缀匹配)** 的冷血逻辑。

## 第三战：静态路由填表与 NAT (网络地址转换) 表手写
**考试权重**：高 (经常和 IP Subnetting 捆绑在一道 18 分大题里)
- **核心概念**：从零手填路由器的三个核心格子：`[目标地址 | 下一跳是谁 | 从哪个端口出去]`；以及 NAT 是如何像个黑市商人一样，篡改 IP 包头里的源 IP (Source IP) 和端口号 (Port) 来骗取公网通信的。
- **我们的任务**：创建 `L3_Static_Routing_and_NAT.md`。
  - 破解 2023/2024 真题里的 NAT 表格全家桶。
  - 搞懂出去 (SNAT) 和进来 (DNAT/Port Forwarding) 为什么要换不同的皮。
  - 死记硬背天下通吃的 Default Route (`0.0.0.0/0`) 填法。

## 第四战：Dijkstra 算法 (Link-State 链路状态) - 绝对防守局
**考试权重**：中等 (虽然在 INHN0012 慢慢变少，但一旦考了画图，如果不按套路走极容易全错)
- **核心概念**：利用全局视角的网络地图，计算出一点到所有点的绝对最短路径树 (Shortest Path Tree)。
- **我们的任务**：创建 `L3_Dijkstra_Link_State.md`。
  - 给出一套考试绝不会错的表格式走格子画法。
  - 理清它和隔壁 RIP (Distance Vector) 的本质区别。
